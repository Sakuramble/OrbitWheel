# OrbitWheel 1.1.0

## 理论支持一切折叠到托盘的应用

- 当软件仍有进程、但不存在正常可见页面时，OrbitWheel 会将其判断为可能已折叠到托盘
- OrbitWheel 会根据动作名称、真实进程名、程序文件描述和 Store AppID 信息匹配 Windows 通知区域中的托盘项
- 找到托盘项后执行受控模拟单击，相当于用户亲自单击该软件的托盘图标
- 使用托盘辅助功能名称匹配，不依赖图标图像；QQ 有通知并更换图标时仍可识别
- 兼容自动隐藏任务栏与高 DPI 缩放
- 从唤出自动隐藏任务栏开始持续锁定鼠标，确认目标软件正常窗口出现后才恢复鼠标位置并解除锁定
- 如果目标软件未能显示窗口，等待 8 秒后自动解除锁定，避免永久限制鼠标
- 软件已显示时直接切换窗口，不触发托盘操作
- Store/AppsFolder 应用会从 AppID 与动作名称解析真实进程名，修复 ChatGPT 等应用被误判为未运行的问题

该机制理论支持所有向 Windows 通知区域正确注册、且单击托盘图标可以恢复窗口的应用。少数不提供名称、使用自定义托盘实现或要求双击/右键菜单的应用可能无法自动匹配。

## 圆环键盘操作

- 从右上扇区开始按顺时针显示 `1–6`，数字位于各扇区靠圆心位置
- 主键盘与小键盘 `1–6` 均可直接执行对应扇区
- 左右方向键可向前或向后切换页面

## 本机验证

- 微信、QQ、ChatGPT、FlClash 关闭至托盘后均成功恢复正常主窗口
- QQ 在通知状态下的托盘名称仍可准确匹配
- 三个软件执行前后进程数量均保持不变
- 自动隐藏任务栏开启时，托盘恢复后鼠标物理坐标保持不变，鼠标锁定已解除
- 8 秒超时压力测试期间连续尝试移动鼠标，均未突破锁定；超时后锁定正常解除

本版本包含此前 OrbitWheel 1.0.x 的全新 UI、图标设计、桌面程序与 Store/UWP 应用运行检测。
